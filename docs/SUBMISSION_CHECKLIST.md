# Kaggle Submission Checklist

## Must Be Done Before Deadline

- [ ] Register / accept rules on Kaggle using one account only.
- [ ] Confirm team size is 1-5 people.
- [ ] Push a public code repository.
- [ ] Deploy a public live demo with no login or paywall.
- [ ] Set `GEMINI_API_KEY` on the deployed app.
- [ ] Verify Gemma 4 workflows run on the deployed app.
- [ ] Record a public YouTube video under 3 minutes.
- [ ] Add a cover image to the Kaggle Media Gallery.
- [ ] Create a Kaggle Writeup under 1,500 words.
- [ ] Select the Impact Track: Digital Equity & Inclusivity.
- [ ] Attach public video, code repo, live demo, and cover image.
- [ ] Press Submit in the Kaggle Writeup before the deadline.

## Technical Verification

- [ ] `npm run typecheck` passes.
- [ ] `npm run build` passes.
- [ ] Public demo can upload PDF/DOCX.
- [ ] Public demo can paste a job description.
- [ ] Public demo can analyze CV.
- [ ] Public demo can generate and download DOCX.
- [ ] Reset button clears the session.
- [ ] No API keys or private CV data are committed.

## Storytelling Checklist

- [ ] Show the real user pain: lack of affordable career coaching.
- [ ] Show the app solving a full workflow end to end.
- [ ] Mention Gemma 4 model names.
- [ ] Show the generated CV, not only the chat UI.
- [ ] Explain why this is digital equity.
